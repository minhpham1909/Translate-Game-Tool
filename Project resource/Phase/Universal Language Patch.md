FEATURE CONTEXT: Universal Language Injector (Export Patch)

1. The Business Problem (Language Selector Dilemma)

When exporting a translation, the user needs a way to switch languages in-game.

Modifying the game's original screens.rpy is extremely dangerous (Regex patching breaks easily on custom UI).

Overriding the preferences screen destroys the developer's custom themes.

Relying on gui.translation_languages is unsafe because custom UI games do not use it.

2. The Architectural Solution: "Floating Keybind Patch"

We will inject a single, standalone .rpy file into the game/ folder during the Export phase.
This file will contain a custom, independent popup screen and bind it to a keyboard shortcut (e.g., Shift + L or F8). It uses Ren'Py's native renpy.known_languages() API to detect translations safely, even inside .rpa archives.

3. The Injection File (vnt_lang_patch.rpy)

The exportService.ts must generate a file named 00_vnt_lang_patch.rpy (the 00_ prefix ensures it loads first) and place it directly into the game/ folder.

Here is the EXACT Ren'Py script to write into that file:

# 00_vnt_lang_patch.rpy - Auto-generated by VN Translator
# Safely injects a universal language switcher without breaking the game's UI.

init python:
    # Safely get all detected languages from the engine
    def vnt_get_available_languages():
        langs = list(renpy.known_languages())
        langs.sort()
        return langs

# The Standalone Popup Screen
screen vnt_language_popup():
    modal True
    zorder 9999 # Ensure it stays on top of everything

    # A simple, universal dark overlay frame
    frame:
        align (0.5, 0.5)
        padding (40, 40)
        background Solid("#1a1a1ae6") # Semi-transparent dark background

        vbox:
            spacing 15
            text "VN Translator - Select Language" size 24 bold True color "#ffffff" xalign 0.5
            null height 10

            # Button for Root/Original Language
            textbutton "Original Game Language":
                action [Language(None), Hide("vnt_language_popup")]
                text_color "#cccccc" hover_text_color "#4ade80"
                xalign 0.5

            # Buttons for dynamically detected translations
            for lang in vnt_get_available_languages():
                textbutton lang.capitalize():
                    action [Language(lang), Hide("vnt_language_popup")]
                    text_color "#cccccc" hover_text_color "#4ade80"
                    xalign 0.5

            null height 20
            textbutton "Close":
                action Hide("vnt_language_popup")
                text_color "#ef4444" hover_text_color "#f87171"
                xalign 0.5

# Bind the popup to a hotkey (F8)
init python:
    config.keymap['vnt_lang_menu'] = ['K_F8']
    config.underlay.append(renpy.Keymap(vnt_lang_menu=lambda: renpy.run(Show("vnt_language_popup"))))

# Add a tiny invisible button to the top right corner (fallback for mouse-only users)
screen vnt_mouse_fallback():
    zorder 9998
    textbutton "🔤":
        text_size 16
        action Show("vnt_language_popup")
        align (1.0, 0.0)
        padding (5, 5)

# Ensure the mouse fallback is always shown
init python:
    config.overlay_screens.append("vnt_mouse_fallback")
